# 汉字 Wordle 计时功能技术方案

## 一、技术实现方案

### 1. 计时启动/停止触发机制

**启动时机**：
- 游戏初始化时（`initGame` 调用）
- 重新开始游戏时（`playAgain` 调用）
- 切换词语时（`changeWord` 调用）

**停止时机**：
- 游戏获胜时（`gameState` 变为 `'won'`）
- 游戏失败时（`gameState` 变为 `'lost'`）
- 用户主动放弃时（点击相关按钮）

**技术实现**：
- 使用 `useEffect` 监听 `gameState` 变化
- 在 `useGame` hook 中添加计时器状态管理
- 实现 `startTimer`、`stopTimer`、`resetTimer` 函数

### 2. 时间计算方法

**核心算法**：
- 使用 `performance.now()` 作为时间戳源（高精度）
- 记录开始时间 `startTime` 和暂停时间 `pauseTime`
- 计算总耗时：`elapsedTime = pauseTime - startTime`

**状态管理**：
```typescript
const [timer, setTimer] = useState({
  startTime: 0,
  pauseTime: 0,
  isRunning: false,
  elapsedTime: 0
});
```

**时间格式化**：
- 转换毫秒为分:秒格式
- 处理两位数显示（00:00 格式）

### 3. 精度保障措施

**精度控制**：
- 使用 `performance.now()` 而非 `Date.now()`（精度从毫秒级提升到微秒级）
- 避免使用 `setInterval`（可能存在计时偏差）
- 使用 `requestAnimationFrame` 进行平滑更新

**误差控制**：
- 误差范围目标：≤ 0.5 秒
- 实现防抖动和去重处理
- 定期同步真实时间戳

**性能优化**：
- 游戏进行中：1000ms 更新一次（秒级精度）
- 游戏结束后：停止更新，保持最终时间

## 三、交互流程设计

### 1. 正常流程

```
游戏初始化 → 计时器启动 → 游戏进行中（计时更新） → 游戏结束 → 计时器停止
```

### 2. 边界情况处理

**页面刷新**：
- 实现本地存储持久化
- 使用 `localStorage` 保存计时状态
- 页面加载时恢复计时

**后台运行**：
- 监听 `visibilitychange` 事件
- 后台时暂停计时
- 前台时恢复计时

**网络中断**：
- 本地计算，不依赖网络
- 网络恢复后正常运行

**多标签页**：
- 使用 `localStorage` 事件同步状态
- 确保只有一个标签页运行计时器

**游戏重置**：
- 点击「再玩一次」：重置计时器并重新开始
- 点击「换一个词」：重置计时器并开始新游戏

## 四、数据整合方案

### 1. 与分享系统对接

**文本分享**：
- 在分享文本中添加时间信息
- 格式：`用时 X分Y秒`
- 位置：结果文本下方

**图片分享**：
- 在分享图片中添加时间信息
- 位置：结果文字下方
- 格式：`用时 X:Y`

**数据传递**：
- 在 `useGame` 中导出 `elapsedTime` 状态
- 分享工具函数接收时间参数
- 统一格式化处理

### 2. 数据格式定义

**时间数据结构**：
```typescript
interface TimerState {
  startTime: number;      // 开始时间戳
  pauseTime: number;      // 暂停时间戳
  isRunning: boolean;     // 是否运行中
  elapsedTime: number;    // 总耗时（毫秒）
}
```

**分享格式**：
- 文本：`用时 ${minutes}分${seconds}秒`
- 图片：`用时 ${minutes}:${seconds}`

## 五、性能与兼容性考量

### 1. 性能优化

**渲染优化**：
- 使用 `useMemo` 缓存时间计算
- 减少不必要的重渲染
- 游戏结束后停止计时更新

**内存管理**：
- 清理定时器和事件监听器
- 避免内存泄漏

**计算优化**：
- 简化时间计算逻辑
- 减少 DOM 操作频率

### 2. 兼容性保障

**浏览器支持**：
- Chrome 60+
- Firefox 55+
- Safari 10.1+
- Edge 79+

**降级方案**：
- 不支持 `performance.now()` 时使用 `Date.now()`
- 不支持 `requestAnimationFrame` 时使用 `setTimeout`
- 不支持 `localStorage` 时使用内存存储

**响应式设计**：
- 适配不同屏幕尺寸
- 移动端优化显示
